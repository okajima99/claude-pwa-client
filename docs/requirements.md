# PWAチャットクライアント 要件定義書

## 1. 背景

外出先（スマートフォン）からAIエージェントにアクセスするために、SSHクライアント（Termius）＋ tmux を使っていた。
機能的には問題ないが、以下のUX課題がある。

- ターミナルUIではメッセージが読みにくい
- 上下スクロールがやりにくい
- フリック入力とターミナルの相性が悪い

これを解消するため、ChatGPTアプリライクなチャットUIをPWAとして自作する。

## 2. 目的

- スマートフォンからAIエージェントと快適にやり取りできる環境を作る
- ターミナル操作を不要にし、チャットUIで完結させる
- 既存のエージェント機能はそのまま維持する

## 3. 想定ユーザー

- 開発者本人のみ（個人利用）

## 4. 解決したい課題

- ターミナルでのメッセージ閲覧・スクロールが辛い
- フリック入力でコマンドを打つのが面倒
- ホーム画面から直接開けるアプリライクな体験が欲しい

## 5. スコープ

### 対象
- 2エージェント（エージェントA / エージェントB）のチャットUI
- セッションの起動・継続・終了
- スマートフォンからのアクセス（Tailscale経由）
- PWA化（ホーム画面追加）

### 対象外
- 複数ユーザー対応
- ユーザー認証・認可
- クラウドデプロイ
- 音声入力UI（OSのキーボード音声入力で代替）

## 6. 機能要件

### 6.1 チャット

- エージェントAタブ／エージェントBタブの切り替えができる
- 両エージェントのセッションが独立して並走する（タブ切り替え中も片方は動き続ける）
- UIからセッション終了（exit相当）を実行できる
- 入力はテキストのみ

### 6.2 セッション継続

- スマートフォンの画面を閉じて戻っても続きから再開できる
- セッション継続はバックエンド側が担保する（フロント接続が切れても母艦のセッションは維持される）

## 7. 非機能要件

### 7.1 アクセス
- Tailscale経由でのプライベートアクセス（インターネット非公開）

### 7.2 可用性
- 母艦で常時稼働するサーバー
- 画面を閉じてもセッションが維持されること

### 7.3 UX
- PWA対応（ホーム画面に追加してアプリライクに使えること）
- チャットUIはメッセージ一覧・入力欄・送信ボタンの基本構成

## 8. アーキテクチャ方針

| 項目 | 方針 |
|---|---|
| フロントエンド | React（PWA） |
| バックエンド | Python（FastAPI） |
| エージェント接続 | `claude --resume` をsubprocessで呼ぶ方式 |
| セッション管理 | `--resume <session-id>` でセッション継続 |
| アクセス経路 | Tailscale経由 |

### 検証が必要な項目

- `--resume` で2エージェントのセッションが独立して管理されるか
- subprocessで呼んだ時の `--resume` の挙動
- セッションIDの保存方法

### 代替案（検証失敗時）

- tmuxで2エージェントのウィンドウを管理し、バックエンドがtmuxを制御する方式

## 9. 実装フェーズ

| フェーズ | 内容 |
|---|---|
| Phase 1 | `--resume` の動作検証・curlで疎通確認 |
| Phase 2 | バックエンドAPI実装（FastAPI） |
| Phase 3 | フロントエンドのチャットUI実装（React） |
| Phase 4 | 2エージェントのタブ切り替え |
| Phase 5 | PWA化（manifest.json / Service Worker） |
